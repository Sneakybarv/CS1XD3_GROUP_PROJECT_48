Initial Client Report

Project Name

Extremity Vault

Client Information

Client Name: Fahtyma Chalhoub
Organizational Affiliation: Independent Artist and Writer

Project Overview

The client would like a web application that serves as a central archive and showcase for a long-term creative passion project. The application will function as both a portfolio and a structured knowledge base, allowing visitors to explore original writing, artwork, and worldbuilding content in an organized and accessible manner.

Target Users

The target users include readers, artists, and creative enthusiasts who are interested in original fictional worlds and storytelling. The application is intended primarily for desktop use, as users are expected to read longer text and view detailed artwork. However, a mobile-friendly format is also desired to ensure accessibility across devices.

Static Information

The web application should provide the following static information:

  • The project name, Extremity Vault
  • The creator's name
  • A link to the creator's art-focused Instagram page

Design and Look & Feel

The client would like the web app to feel artistic, creative, and in tune with the fictional world being presented. At the same time, the design should remain minimalist and well organized so users are not overwhelmed. Clear navigation, intentional layouts, and a calm visual style are preferred, while excessive clutter or distracting elements should be avoided.

Data to Be Managed or Presented

The application should manage and present:

  • Wiki-style articles containing character information, worldbuilding details, and plot summaries
  • An organized catalogue of original writing and artwork
  • Interlinked content that allows users to move easily between related information
  • User-submitted creative work, pending approval by administrators
  • Optional user accounts for saving data and personal content

Interactive and Media Elements

The web app should support embedded images and videos to showcase artwork and related media. It should also provide a comfortable reading experience for written content. While no complex interactive features are required initially, smooth navigation and content access are essential.
